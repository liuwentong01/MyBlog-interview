## 一、项目概述

### 1.1 背景
- **场景**：入门班班主任与学员/家长电话沟通后，通过AI模型对沟通内容进行分析总结
- **核心目标**：为班主任提供沟通回顾、用户分层、转化建议；为管理人员提供质检与策略调整抓手

### 1.2 需求单
- YBCXQGL-7351：入门班-AI沟通总结
- YBCXQGL-7638：沟通总结分层规则调整与风险转化标签化

### 1.3 覆盖范围
- 级别限制：C入/R入
- 灰度方式：根据ldap或组织结构覆盖

## 二、不同阶段的使用场景

| 阶段 | 班主任使用 | 管理人员使用 |
|------|-----------|-------------|
| **种草期** | 收集数据为主 + 沟通内容框架引导 | 质检班主任执行情况 |
| **行课期** | 用户分层 + 沟通前回顾 + 制定针对性方案 | 质检 + 根据学员情况调整业务动作 |
| **转化期** | 用户分层 + 沟通回顾 + 寻求沟通建议与风险评估 | 质检 + 调整业务动作 |

### 管理人员具体使用方式
1. **质检**：根据AI沟通总结评估班主任沟通质量，给出指导建议
2. **质检**：根据总结反馈内容项数量，评判班主任沟通做工深度
3. **分析**：观察学员分层等级变化，调整SOP执行策略

## 三、功能模块：带班列表

### 3.1 基本信息
- **位置**：我的带班 → 沟通类目下，QT标签后面
- **覆盖模块**：课前沟通、课后回访、转化沟通三个tab均增加
- **数据更新**：从班课分班开始至班课结束+1，每天3:00更新前一天数据
  - 限制：前一天无新增沟通数据则不更新（排除班主任自身的沟通数据）

### 3.2 通用规则
1. 所有模型未返回或未能解析的内容均返回"未知"
2. **模型幻觉处理**：新生成的"学员分析"维度数据如果未包含历史已生成的维度数据，需将历史数据补充显示（并集逻辑）
3. AI总结只显示班课对应**最后带班班主任**与学员的沟通总结
4. 数据存在层级关联关系

### 3.3 筛选功能

筛选项之间为"且"逻辑，底层数据来自大模型反馈：

| 筛选项 | 类型 | 枚举值 |
|--------|------|--------|
| 编程基础 | 多选 | 有、无、未知 |
| 设备环境 | 多选 | 满足、不满足、未知 |
| 上课时间确认 | 多选 | 确认、未确认 |
| 能力差异 | 多选 | 思维能力弱、专注力差、自信心不足、学科成绩差、未知 |
| 核心诉求 | 多选 | 思维能力提升、专注力提升、自信心提升、学科成绩提升、促进升学、兴趣培养、未知 |
| 赛考/考级/升学 | 多选 | 了解、未了解、未知 |
| 消费能力 | 多选 | 高、中、低、未知 |

| 用户分层 | 多选 | S+、S、A、B、未知 |
| 沟通"学情分析"总结项数 | 升序/降序 | 按项目数量 |
| 对比昨日新增沟通总结项数 | 升序/降序 | 差值数量 |

**权限限制**：
- 配合程度、用户分层：带班班主任U1当天0点后才显示，其他角色始终可见
- 昨日对比筛选项：班课结束+1后隐藏

**用户分层包装**：原始等级S/A/B/C → 显示为S+/S/A/B

## 四、AI沟通总结-分析报告（浮层详情）

### 4.1 基础功能
- **生成时间**：显示AI模型最后一次生成时间（年-月-日 时:分）
- **更新按钮**：手动重新调用大模型
  - 生成中：文案改为"生成中"，图标动态
  - 关闭再打开浮层：延续生成中状态
  - 生成失败：恢复初始状态
  - **限制条件**：
    - 班课结课超过15天 → 禁用，hover："班课结束后超过15天，不能进行AI总结"
    - 当天手动更新超过3次 → 禁用，hover："每日更新使用次数限制3次"
- 无数据时显示"（信息不足，无法判断）"

### 4.2 解析数据Tab

#### 模块1：AI分析-学员分析

**数据特点**：累积多日生成数据的**并集**（仅学员分析取并集，其他子模块取最终结果）

**子模块结构**：

**基础能力深度分析**
- 编程基础 [单标签]：有/无/未知（hover显示判断依据）
- 能力差异 [多标签]：思维能力弱/专注力差/自信心不足/学科成绩差/未知（hover逐个显示判断依据）

**家长核心需求与规划分析**
- 核心诉求 [多标签]：思维能力提升/专注力提升/自信心提升/学科成绩提升/促进升学/兴趣培养/未知
- 赛考/考级/升学 [单标签]：了解/未了解/未知

**具备上课条件分析**
- 上课时间确认 [单标签]：确认/未确认/未知
- 设备环境 [单标签]：满足/不满足/未知

**其他子模块**（统一格式：标签+描述）
- 过往学习与课程适配分析
- 兴趣偏好与学习体验分析
- 家庭背景与支持条件分析
- 地域与特殊场景适配分析

**高亮规则**：今天生成的标签如果是前一天没有的，高亮显示

**未返回数据处理**：相关维度依然显示，内容显示"沟通未涉及"

#### 模块2：AI分析-转化分析

**转化分析内容**：
- 消费能力 [单选]：高/中/低
  - 用户等级描述
  - 邮寄地址房价参考（通过邮寄地址获取，无地址不显示）
  - 设备型号参考价（通过登录信息获取，无信息不显示）
  - 兴趣班（价格信息模型返回）
- 沟通与配合程度 [单选]：高/中/低
  - 配合程度描述
- 转化风险点 [文本]
- 关键决策因素 [文本]

**用户分层**：
- 用户等级 [单选]：S+/S/A/B（原始S/A/B/C包装后）
- 用户等级描述（如"潜力待激发型"）
- 判定依据：0到多个，由模型自行判断

#### 模块3：AI分析-关键内容提炼
- 数据格式：标签+描述
- 模型未返回则不显示

### 4.3 原始数据Tab
- 显示模型原始输出内容
- AI总结版本 [下拉单选]：按日期倒序，默认最新版本
- **权限管理**：仅具备权限的人员可见原始数据（无权限隐藏解析数据和原始数据tab）

## 八、数据解析-无效内容过滤规则

### 问题描述
模型对沟通未涉及的内容会额外回复（幻觉），如"无（沟通中未提及任何过往编程学习经历）"

### 产品决策
**以敏感度为主**（追求覆盖而非严格过滤）

### 过滤规则

**规则1**：输出内容小于4个文字 → 判定无效

**规则2**：包含特定关键词 → 判定无效
- **模糊匹配**：未提及、未明确、无法判断、信息不足、未明确提及、暂无法判断、无明确信息、未明确表达、无相关信息、尚未正式上课、尚未上课
- **绝对匹配**：无

**边界处理**：模型反馈"无（未提及）"时，关键词判断的是括号前的内容

### 子模块级容错
- 某个子模块内容不符合格式时，该子模块不解析，但其他模块正常解析
