> 来源：Confluence PRD，需求负责人：初锋，最后更新：2025-12-08

## 一、项目概述

### 1.1 背景
- **场景**：郑州系统班班主任老师在每次关键电话沟通后，需手动总结通话内容并发送给家长
- **痛点**：人力占用高、总结不全面、内容不聚焦
- **方案**：通过AI大模型自动分析通话录音，生成结构化沟通总结供班主任使用

### 1.2 价值
- 提升人效：省去人工总结成本
- 数据价值：提供更全面、精准的分析数据

### 1.3 灰度范围
- 一期灰度：郑州系统班（宋子毅管理范围的所有团队）
- 无权限系统控制，所有班主任老师均可使用

## 四、核心功能：通话总结列表

### 4.1 数据范围
- 班主任与学员的所有外呼沟通数据
- 数据来源：坐席外呼 / 企微音视频 / 工作手机外呼
- 学员归属通过微信绑定关联获取
- 排序：按外呼拨打时间正序排列

### 4.2 列表元素
- 外呼拨打时间（年.月.日 时:分）
- 接通状态（已接通 / 未接通）
- 外呼渠道类型（坐席外呼 / 企微音视频 / 手机外呼）
- 总结状态（未总结 / 总结中 / 已总结 / 总结失败）
- 展开/收起（默认收起）

### 4.3 兜底场景
| 场景 | 处理 |
|------|------|
| 学员微信未绑定猿编程ID | 显示通用兜底页面，绑定后刷新 |
| 无通话记录 | 显示"暂无通话记录！" |
| AI模型整体停用 | 显示"AI工具维护调整中，暂停使用！" |

## 五、核心功能：AI内容生成（子模块）

### 5.1 通用能力 - 语音转文字
- 音频播放组件 + "转文字"按钮
- 有转化数据：展开显示沟通记录内容，隐藏按钮；再次进入恢复初始状态
- 未转化：禁用状态，hover提示"转文字内容生成中"
- 沟通内容为空时显示兜底文案

### 5.2 场景状态机（8个场景）

```
场景1: 未通话 + 未总结 → 不可操作AI
场景2: 已通话 + 未总结 → 可点击"生成AI通话总结"
场景3: 已通话 + 总结中 → 工作流逐步执行（实时反馈）
场景4: 已通话 + 已总结 → 展示AI推荐结果
场景5: 已通话 + 执行失败 → 显示失败信息，可重新生成
场景6: 已通话 + 已总结但沟通内容为空 → 兜底样式
场景7: 已通话 + 非工作手机外呼 → 不可操作（未安装emon）
场景8: 已通话 + 语音数据未同步 → 可操作，等待数据同步
```

## 六、工作流详细设计

### 6.1 工作流步骤（5步）

```
步骤1: 外呼语音回传 → 语音文件同步到QT系统
步骤2: 语音转换文字 → 通话语音转化为文本
步骤3: 文字脱敏处理 → 文本内容脱敏（参考AI风控规则）
步骤4: AI模型分析   → 脱敏文本送入AI模型分析总结
步骤5: 解析分析结果 → 解析为4个模块内容
```

### 6.2 步骤节点状态
- 待执行 → 执行中 → 已执行 / 执行失败
- 顺序执行，前一步完成后才进入下一步
- 任一步骤失败则整体终止
- 页面实时显示执行进度和耗时（秒）

### 6.3 关键规则
- **步骤1超时**：超过20分钟未获取语音文件 → 判定失败
- **步骤2预处理**：语音转文本可预先执行，操作AI总结时判断是否已完成（已完成直接使用，未完成每10秒轮询）
- **切页恢复**：切换页面再回来，执行状态和计时需延续

### 6.4 完成通知
- 成功：企微Qwerty助手发送 `年/月/日 时:分_AI通话总结已生成（学员id-学员昵称-微信昵称）`
- 失败：发送 `异常通知：年/月/日 时:分_AI通话总结已生成"失败"（学员id-学员昵称-微信昵称）`

## 七、AI输出结果结构（4个模块）

### 模块1: 电话沟通留言（推荐话术）
- **格式**：`电话沟通留言：[留言类型]：判断依据` + 编号话术列表
- **解析规则**：
  - 非兜底类型：显示留言类型标签（hover显示判断依据），按编号排序展示话术
  - 兜底类型：直接显示兜底内容
- **操作能力**：
  - 批量发送（一键发送所有话术子模块）
  - 单条发送 / 单条复制到微信输入框
  - 发送前校验Emon绑定状态
- **发送状态**：发送中 → 发送成功/发送失败，超时30秒转失败
- **话术支持展开/收起显示**

### 模块2: 提炼关键内容
- 原文展示，如：学习进度、学习状态、技术问题、课后安排等

### 模块3: 家长期望
- 原文展示

### 模块4: 家长配合程度
- 原文展示

## 八、重新生成机制

### 8.1 模型输出不满意 → 重新生成
- 最多3次
- 用户填写问题描述（必填，文本输入）
- 提示词：`反馈内容有如下问题，请调整后重新生成内容：【问题内容】`
- **关键**：重新生成需在同一AI会话中（联动上下文）
- 超过3次按钮禁用，hover提示"重新生成次数限制3次"

### 8.2 工作流失败 → 重新生成
- 最多3次（与上述分开计数）
- 从失败节点继续执行，非重头开始
- 区分首次分析（新会话）和重新生成中失败（延用原会话）

## 十、三期需求：阿里云语音服务 + 满意度评价

### 10.1 阿里云语音转文字
- **问题**：AILab语音转文字响应慢（3-10分钟，高峰不确定）
- **方案**：接入阿里云语音识别服务（30分钟音频10秒内完成）
- **调用逻辑**：AILab未完成时，调用阿里云服务
- **数据优先级**：阿里云数据优先于AILab数据
- **微信语音/视频渠道**：还需额外调用"说话人识别"做人员区分

### 10.2 语音转文字交互调整（三期）
| 渠道 | 未转化时交互 |
|------|-------------|
| QT坐席 | 显示"转化中" + 等待标签，完成后显示内容；异常时显示"转文字" |
| 手机/微信 | 禁用状态，hover提示"转文字内容生成中" |

### 10.3 满意度评价
- 推荐内容满意度：差评 / 中评 / 好评（单选）
- 改进建议：多行输入框，非必填，最大500字
- 提交后不可修改，再次进入评价模块收起

## 十三、业务Prompt设计要点

### 核心要求
1. **风格**：像朋友聊天一样自然，有温度，不机械，不分点列举
2. **结构**：沟通重点 + 接下来怎么做
3. **身份**：保持老师身份，提醒作业用引导而非催促
4. **格式**：连贯的一段文字，不用冒号、编号
5. **标签**：通俗易懂，避免专业术语

### 三个输出任务
1. **留言文案**：基于通话内容生成可发送给家长的文字留言
2. **关键内容提炼**：精准到细节的短句，用于填写跟进表
3. **标签生成**：判断家长对孩子学编程的意向原因
